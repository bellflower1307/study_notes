    継承

    ここからはオブジェクト指向プログラミングとして継承について学びます。

    Javaでは、既存のクラスを基に新しいクラスを定義することができ、これによりコードの再利用が可能になります。この仕組みを継承といいます。

    継承を行うことで、親クラスのフィールドとメソッドを子クラスで使用できるようになります。

    継承の基本的な構文は以下の通りです。

    [修飾子] class 子クラス名 extends 親クラス名 {
    // 子クラスのフィールドやメソッド
}

    ここで、extendsキーワードを使用して、子クラスが親クラスを継承することを示しています。

    継承を行うと、private修飾子がついていない親クラスのフィールドとメソッドは子クラスに引き継がれます。また、子クラスは親クラスの機能を拡張したり、新しい機能を追加したりできます。

    継承の例を見てみましょう。

    class Animal {
    void eat() {
        System.out.println("This animal eats food.");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("The dog barks.");
    }
}

    この例では、Animalクラスが親クラスであり、Dogクラスが子クラスです。DogクラスはAnimalクラスを継承しているため、eat()メソッドを使用できます。

    継承を使用することで、コードの再利用が促進され、プログラムの構造がより明確になります。

    親クラスのことをスーパークラス、子クラスのことをサブクラスと呼びます。

    extendsキーワード後に指定する親クラスは、1つしか指定できません。複数指定するとコンパイルエラーになります。

    また、すべてのクラスはjava.lang.Objectクラスを暗黙的に継承しています。

    以下では、継承の具体的な使い方について説明します。

    オーバーライド

    子クラスは、親クラスのメソッドをオーバーライドすることができます。オーバーライドとは、親クラスのメソッドを子クラスで再定義することです。

    class Dog extends Animal {
    @Override
    void eat() {
        System.out.println("The dog eats dog food.");
    }
}

    この例では、DogクラスがAnimalクラスのeat()メソッドをオーバーライドしています。オーバーライドされたメソッドは、子クラスのインスタンスで呼び出されると、子クラスの実装が実行されます。

    オーバーライドの条件として、以下があります。

    • メソッド名が同じであること
    • 引数リストの型と数が同じであること
    • 戻り値の型が同じかスーパークラスで定義した型のサブクラスであること
    • アクセス修飾子が同じか、より広いアクセスレベルであること
    • 例外の宣言が同じか、より少ない例外を宣言していること
    • static修飾子が同じであること
    • final修飾子がついていないこと

    例外については、例外処理の章で詳しく説明します。

    オーバーライドでは、インスタンスメソッドをstaticメソッドとしてオーバーライドすることはできません。

    スーパークラスで定義されたfinal修飾子がついたメソッドは、オーバーライドできません。

    Overrideアノテーション

    Overrideアノテーションは、メソッドがオーバーライドされることを示すために使用されます。

    Overrideアノテーションが付与されたメソッドがオーバーライドされていない場合、コンパイラはエラーを報告します。

    このアノテーションを使用することで、メソッド名の誤りなどによるオーバーライド漏れがあった場合、コンパイラはオーバーライドのエラーとして検出できます。

    superキーワード

    superキーワードは、子クラスから親クラスのメンバーにアクセスするために使用されます。

    superキーワードを使用して、親クラスのメソッドを呼び出すことができます。

    class Dog extends Animal {
    @Override
    void eat() {
        super.eat(); // 親クラスのeat()メソッドを呼び出す
        System.out.println("The dog eats dog food.");
    }
}

    この例では、super.eat()を使用して、親クラスのeat()メソッドを呼び出しています。

    superキーワードは、親クラスのメソッドだけではなくフィールドにアクセスするためにも使用できます。

    自分自身のフィールドにアクセスするには、thisキーワードを使用します。

    継承とコンストラクタ

    子クラスのコンストラクタは、親クラスのコンストラクタを呼び出す必要があります。これを行うために、子クラスのコンストラクタの最初の行でsuper()を使用します。

    class Dog extends Animal {
    Dog() {
        super(); // 親クラスのコンストラクタを呼び出す
    }
}

    この例では、super()を使用して、親クラスのコンストラクタを呼び出しています。親クラスのコンストラクタが引数を取る場合は、必要な引数をsuper()に渡す必要があります。

    コンストラクタにsuper()が指定されていない場合、コンパイラは暗黙的にsuper()を挿入します。ただし、親クラスに引数を取るコンストラクタがある場合は、明示的にsuper()を呼び出す必要があります。

    また、コンストラクタにthis()やsuper()を記載する場合は、必ず最初の行に記載する必要があります。2行目以降に記載するとコンパイルエラーになります。

    
class Dog extends Animal {
    Dog() {
        this("Default Name"); // this()はコンストラクタの最初の行でなければならない
    }
    Dog(String name) {
        super(); // super()もコンストラクタの最初の行でなければならない
    }
    Dog( String name, int age) {
            super(); // これはコンストラクタの最初の行なのでOK
            this("Default Name"); // これは2行目以降なのでコンパイルエラー
    }
}

    参照型の型変換

    継承関係にあるクラスのオブジェクトは、親クラスの型で参照することができます。これをアップキャストといいます。

    Animal animal = new Dog(); // アップキャスト

    この例では、DogクラスのオブジェクトをAnimalクラスの型で参照しています。アップキャストは暗黙的に行われるため、明示的なキャストは必要ありません。

    一方、親クラスの型で参照しているオブジェクトを子クラスの型に変換することもできます。これをダウンキャストといいます。

    Dog dog = (Dog) animal; // ダウンキャスト

    ダウンキャストは明示的に行う必要がありますが、実行時にオブジェクトが実際に子クラスの型であることが保証されていない場合、ClassCastExceptionがスローされる可能性があります。

    ダウンキャストを安全に行うためには、instanceof演算子を使用して、オブジェクトが特定のクラスのインスタンスであるかどうかを確認することができます。

    if (animal instanceof Dog) {
    Dog dog = (Dog) animal; // 安全なダウンキャスト
}

    アップキャストを行った場合、オーバーライドしたメソッドを呼び出す場合は、以下の注意が必要です。

    • インスタンスメソッドを呼び出す場合、子クラスの実装（キャストされる前の型）が呼び出されます。
    • staticメソッドを呼び出す場合、親クラスの実装（キャストされた型）が呼び出されます。
    • フィールドを参照する場合、親クラスの実装（キャストされた型）が参照されます。
    
Animal animal = new Dog(); // アップキャスト
animal.eat(); // インスタンスメソッドは子クラスの実装が呼び出される
Animal.staticMethod(); // staticメソッドは親クラスの実装が呼び出される
System.out.println(animal.field); // フィールドは親クラスの実装が参照される

    複数クラスで継承関係があった場合も同様です。AnimalクラスのサブクラスとしてDogクラスがあり、DogクラスのサブクラスとしてBreedクラスがある場合も同様です。

    
Dog dog = new Breed(); // アップキャスト
Animal animal = dog; // アップキャスト
animal.eat(); // インスタンスメソッドはBreedクラスの実装が呼び出される

    この例では、BreedクラスがDogクラスを継承し、DogクラスがAnimalクラスを継承しています。アップキャストを行った場合でも、インスタンスメソッドは実際のオブジェクトのクラスの実装が呼び出されます。

    参照型で型変換がある場合は、変数宣言時の型と、実際のオブジェクトの型の2つを意識しておきましょう。
