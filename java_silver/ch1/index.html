    Javaの基礎

    この章では、Javaの基本的な特徴と、プログラムの実行方法について学びます。

    Javaの特徴

    Javaには以下の特徴があります。

    JVM (Java Virtual Machine)

    JavaはJVM(Java 仮想マシン)を介して実行されます。

    Cのような言語では、コンパイル時に機械語に変換されていました。機械語はそれぞれのOSに依存しているため、あるOSでコンパイルされたプログラムは他のOSでは実行できません。

    一方、Javaはコンパイル時にバイトコードと呼ばれる中間コードに変換されます。

    変換されたバイトコードは、JVMによって実行されます。このJVMはOSに依存しないため、同じバイトコードを異なるOSで実行できます。

    このような性質は「Write Once, Run Anywhere（WORA）」と呼ばれています。

    オブジェクト指向

    オブジェクト指向とは、プログラムの中で扱う対象をオブジェクトとして扱うプログラミング手法です。

    Javaでは、オブジェクト指向の概念を活用して、より効率的で保守しやすいプログラムを構築できます。

    ガーベジコレクション

    Javaはガーベジコレクションと呼ばれる自動メモリ管理機能を備えています。

    ガーベジコレクションは、プログラムが使用しなくなったメモリを自動的に解放する機能です。

    これにより、プログラマはメモリ管理の負担を軽減でき、メモリリークやダングリングポインタなどの問題を回避できます。

    Java環境

    JDK (Java Development Kit)

    Javaを利用してプログラムを作成したり、作成したプログラムを実行するにはJava開発環境が必要です。

    Java開発環境はJDK(Java 開発キット)で構築します。

    JDKはOracle社だけではなく、様々なベンダーから提供されており、有償のものや無償のものがあります。

    Javaのバージョン

    Javaは複数のバージョンが存在し、それぞれに異なる機能や改善が含まれています。

    年6回の定期リリースとして、毎年3月と9月に新しいバージョンがリリースされます。

    JRE (Java Runtime Environment)

    JRE(Java 実行環境)はJavaアプリケーションを実行するために必要な環境を提供します。

    Java 11から単体配布されなくなり、JDKに含まれるようになりました。

    また、Java環境を持たないユーザのために、配布するアプリケーションにJREが同梱されるようになりました。

    Javaプログラムのコンパイルと実行

    Javaプログラムは以下の手順で作成・実行されます。

    1. Javaソースコードをテキストエディタで作成し、.javaファイルとして保存する。
    2. コマンドラインでjavacコマンドを使用して、.javaファイルをコンパイルし、.classファイルを生成する。
    3. javaコマンドを使用して、生成された.classファイルをJVMで実行する。

    ソースファイルの作成

    Javaソースコードは、テキストエディタを使用して作成されます。

    ソースコードは.javaの拡張子を持つファイルに保存されます。

    コンパイル

    Javaソースコードは、javacコマンドを使用してコンパイルされます。

    javac {ソースファイル名}.java

    コンパイルに成功すると、同じ名前の.classファイルが生成されます。

    ソースコードに文法上の誤りが存在する場合、コンパイル時にコンパイルエラーが発生します。

    実行

    生成された.classファイルは、javaコマンドを使用してJVMで実行されます。

    java {クラス名}

    コンパイルに成功したクラスファイルが正しくない実行を行った場合、実行時エラーが発生します。

    また、一定の条件下ではコンパイルをせずに実行することもできます（後述）。

    ソースファイルとクラスファイル

    ソースファイルには、Javaのクラスやメソッドなどのコードが記述されます。

    クラスファイルの名称はソースファイル内に書かれたクラス名と一致します。

    一つのソースファイルに複数のクラスを含めることができ、その場合はコンパイル時にすべてのクラスのクラスファイルが作成されます。

    ただし、publicクラスは一つのソースファイルに一つしか記述できず、ソースファイル名とpublicクラスの名前が一致する必要があります。

    また、CLASSPATH上にクラスファイルがない場合、コンパイルせずに実行することもできます。その場合はソースファイルの最初に記載されているクラスが実行されます。

    java {ソースファイル名}.java

    パッケージ宣言とインポート

    Javaでは様々なクラスが存在しており、それらをまとめたものをパッケージと呼びます。

    パッケージ化するには、ソースファイルの最初に"package"キーワードを使用してパッケージを宣言します。

    package {パッケージ名};

    他のパッケージからクラスを呼び出すには、"import"キーワードを使用します。

    import {パッケージ名}.{クラス名};
    import {パッケージ名}.*;

    また、"import"キーワードを使用せずにクラスを呼び出すこともできます。その場合はクラス名の前にパッケージ名を付ける必要があります。

    {パッケージ名}.{クラス名}

    "package"はソースファイルの先頭に記載する必要があり、1つまで記載できます。

    "import"は"package"の次に記載する必要があり、複数記載できます。"package"を記載していない場合はソースファイルの先頭に記載できます。

    記載の順序は、"package"が最初に、その後に"import"が続く必要があります。

    Javaが提供する主なパッケージには以下があります。

    • java.lang: Javaの基本的なクラスが含まれるパッケージ
    • java.util: コレクションフレームワークや日付・時間などのユーティリティクラスが含まれるパッケージ
    • java.io: 入出力に関するクラスが含まれるパッケージ
    • java.net: ネットワーク通信に関するクラスが含まれるパッケージ
    • java.awt: GUIアプリケーションのためのクラスが含まれるパッケージ
    • javax.swing: より高度なGUIアプリケーションのためのクラスが含まれるパッケージ

    java.langのみimport文がなくても自動でインポートされます。
