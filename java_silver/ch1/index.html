    Javaの基礎

    この章では、Javaの基本的な概念と特徴について学びます。

    Javaの特徴

    Write Once, Run Anywhere (WORA)

    JavaはプログラムとOSの間に依存関係がないため、一度書いたコードを様々な環境で実行できます。

    JVM (Java Virtual Machine)

    JavaはJVM(Java 仮想マシン)を介して実行されます。

    オブジェクト指向

    Javaはオブジェクト指向プログラミング言語であり、クラスとオブジェクトを使用してコードを構築します。

    Java環境

    JDK (Java Development Kit)

    Javaを利用してプログラムを作成したり、作成したプログラムを実行するにはJava開発環境が必要です。

    Java開発環境はJDK(Java 開発キット)で構築します。

    JDKはOracle社だけではなく、様々なベンダーから提供されており、有償のものや無償のものがあります。

    Javaのバージョン

    Javaは複数のバージョンが存在し、それぞれに異なる機能や改善が含まれています。

    年6回の定期リリースとして、毎年3月と9月に新しいバージョンがリリースされます。これをフィーチャー・リリースと呼びます。

    また、各フィーチャー・リリースには、新しい言語機能やパフォーマンスの向上、セキュリティの強化などを施したアップデート・リリースが2回含まれます。

    JRE (Java Runtime Environment)

    JRE(Java 実行環境)はJavaアプリケーションを実行するために必要な環境を提供します。

    Java 11から単体配布されなくなり、JDKに含まれるようになりました。

    また、Java環境を持たないユーザのために、配布するアプリケーションにJREが同梱されるようになりました。

    Javaプログラムのコンパイルと実行

    Javaプログラムは以下の手順で実行される。

    1. Javaソースコードをテキストエディタで作成し、.javaファイルとして保存する。
    2. コマンドラインでjavacコマンドを使用して、.javaファイルをコンパイルし、.classファイルを生成する。
    3. javaコマンドを使用して、生成された.classファイルをJVMで実行する。

    ソースコードに文法上の誤りが存在する場合、コンパイル時にコンパイルエラーが発生する。

    コンパイルに成功したクラスファイルが正しくない実行を行った場合、実行時エラーが発生する。

    例えば、main()メソッドでは以下がコンパイルエラーと実行時エラーになる。

    1. コンパイルエラー：main()メソッドの引数の型がstringである場合
      2. public static void main(string arg) { ... }
    2. コンパイルエラー：main()メソッドの戻り値がない場合
      3. public static main(String[] args) { ... }
    3. コンパイルエラー：main()メソッドの引数名がない場合
      4. public static void main(String[]) { ... }
    4. 実行時エラー：main()メソッドの引数がString配列ではなくStringの場合
      5. public static void main(String arg) { ... }
    5. 実行時エラー：main()メソッドにstaticがない場合
      6. public void main(String[] args) { ... }
    6. 実行時エラー：main()メソッドがMain()の場合
      7. public static void Main(String[] args) { ... }

    コンパイルは以下のコマンドを用いる。

    javac {ソースファイル名}.java

    実行は以下のコマンドを用いる。

    java {クラス名}

    また、一定の条件下ではコンパイルをせずに実行することもできます（後述）。

    ソースファイルとクラスファイル

    メモ帳などのテキストエディタで記述したプログラムファイルをソースファイルと言います。

    ソースファイルは.javaの拡張子を持ち、コンパイルすることで.classファイルが作成されます。

    クラスファイルの名称はソースファイル内に書かれたクラス名と一致します。

    一つのソースファイルに複数のクラスを含めることができ、その場合はコンパイル時にすべてのクラスのクラスファイルが作成されます。

    ただし、publicクラスは一つのソースファイルに一つしか記述できず、ソースファイル名とpublicクラスの名前が一致する必要があります。

    また、CLASSPATH上にクラスファイルがない場合、コンパイルせずに実行することもできます。その場合はソースファイルの最初に記載されているクラスが実行されます。

    java {ソースファイル名}.java

    パッケージ宣言とインポート

    Javaでは様々なクラスが存在しており、それらをまとめたものをパッケージと呼びます。

    パッケージ化するには、ソースファイルの最初に`package`キーワードを使用してパッケージを宣言します。

    package {パッケージ名};

    他のパッケージからクラスを呼び出すには、`import`キーワードを使用します。

    import {パッケージ名}.{クラス名};
    import {パッケージ名}.*;

    また、`import`キーワードを使用せずにクラスを呼び出すこともできます。その場合はクラス名の前にパッケージ名を付ける必要があります。

    {パッケージ名}.{クラス名}

    packageはソースファイルの先頭に記載する必要があり、1つまで記載できます。

    importはpackageの次に記載する必要があり、複数記載できます。packageを記載していない場合はソースファイルの先頭に記載できます。

    記載の順序は、packageが最初に、その後にimportが続く必要があります。

    Javaが提供する主なパッケージには以下があります。

    • java.lang: Javaの基本的なクラスが含まれるパッケージ
    • java.util: コレクションフレームワークや日付・時間などのユーティリティクラスが含まれるパッケージ
    • java.io: 入出力に関するクラスが含まれるパッケージ
    • java.net: ネットワーク通信に関するクラスが含まれるパッケージ
    • java.awt: GUIアプリケーションのためのクラスが含まれるパッケージ
    • javax.swing: より高度なGUIアプリケーションのためのクラスが含まれるパッケージ

    java.langのみimport文がなくても自動でインポートされます。
