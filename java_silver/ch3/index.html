    式と演算子

    この章では、Javaでの式と演算子について学びます。

    式

    式とは、値を生成するコードのことです。式は、値を返すことができます。例えば、以下のコードは式の例です。

    int x = 5 + 3; // 5 + 3は式で、8という値を生成します。

    式は、演算子や関数呼び出しなどを含むことができます。例えば、以下のコードは複雑な式の例です。

    int y = (x * 2) + Math.max(10, 20); // (x * 2) + Math.max(10, 20)は式で、xの値に基づいて値を生成します。

    関数呼び出しを除くと、式は演算子とリテラルから構成されます。演算子によって操作されるリテラルや変数のことをオペランドと呼びます。

    演算子

    演算子は、式の中で値を操作するために使用されます。Javaには、以下の種類の演算子があります。

    • 算術演算子
    • 比較演算子
    • シフト演算子
    • 論理演算子
    • 代入演算子
    • 三項演算子
    • その他の演算子（条件演算子、インスタンスチェック演算子など）

    演算子は、オペランドの型に応じて異なる動作をすることがあります。例えば、+演算子は数値の加算に使用されますが、文字列の結合にも使用されます。

    また、演算子には優先順位があり、複数の演算子が同じ式に含まれる場合、優先順位に従って評価されます。必要に応じて、括弧を使用して評価の順序を明示することもできます。

    以下では各演算子について詳しく説明します。

    算術演算子

    算術演算子は、数値の計算に使用されます。Javaの算術演算子には、以下のものがあります。

    • 加算（+）
    • 減算（-）
    • 乗算（*）
    • 除算（/）
    • 剰余（%）
    • インクリメント（++）
    • デクリメント（--）

    算術演算子は、数値の計算に使用されますが、文字列の結合にも使用されることがあります。例えば、以下のコードは文字列の結合に+演算子を使用しています。

    String str1 = "Hello, ";
String str2 = "World!";
String str3 = str1 + str2; // str1 + str2は式で、"Hello, World!"という値を生成します。

    また、インクリメントとデクリメントには前置と後置で異なる動作をします。

    int a = 5;
int b = a++; // 後置インクリメント: bは5、aは6
int c = ++a; // 前置インクリメント: aは7、cは7

    このように、前置の場合は変数の値が先に増加し、その後代入処理が入ります。一方で後置の場合は、代入処理が先に実行され、その後変数の値が増加します。

    比較演算子

    比較演算子は、値を比較するために使用されます。Javaの比較演算子には、以下のものがあります。

    • 等しい（==）
    • 等しくない（!=）
    • 大なり（>）
    • 小なり（<）
    • 大なりイコール（>=）
    • 小なりイコール（<=）

    比較演算子は、値を比較して真(true)または偽(false)の論理値を返します。例えば、以下のコードは比較演算子の例です。

    int x = 5;
boolean y = (x == 5); // yはtrue
boolean z = (x != 5); // zはfalse

    シフト演算子

    シフト演算子は、ビット単位でのシフト操作に使用されます。Javaのシフト演算子には、以下のものがあります。

    • 左シフト（<<）
    • 右シフト（>>）
    • 論理右シフト（>>>）

    シフト演算子は、ビットを左または右にシフトするために使用されます。例えば、以下のコードは左シフト演算子の例です。

    int x = 5; // 00000000 0000 0000 0000 0000 0000 0101
int y = x << 1; // yは10 (00000000 0000 0000 0000 0000 0000 1010)

    シフト演算子は、ビット単位の操作に使用されるため、数値の計算とは異なる動作をします。シフト演算子を使用する際には、ビットの表現や符号に注意する必要があります。

    論理演算子

    論理演算子は、論理値を操作するために使用されます。

    論理演算子にはサーキット演算子とビット演算子の2つがあります。

    サーキット演算子

    サーキット演算子は、論理値を操作するための演算子で、短絡評価を行います。短絡評価とは、左辺の値だけで結果が確定する場合に、右辺の評価を行わないことを指します。例えば、論理積（&&）の場合、左辺がfalseであれば、右辺の評価は行われません。

    サーキット演算子は、条件分岐やループの条件式などでよく使用されます。短絡評価を利用することで、不要な計算を避けることができ、プログラムの効率を向上させることができます。

    Javaのサーキット演算子には、以下のものがあります。

    • 論理積（&&）
    • 論理和（||）
    • 論理否定（!）

    論理演算子は、論理値を操作して新しい論理値を生成します。例えば、以下のコードは論理演算子の例です。

    boolean a = true;
boolean b = false;
boolean c = a && b; // cはfalse
boolean d = a || b; // dはtrue
boolean e = !a; // eはfalse
    ビット演算子

    ビット演算子は、ビット単位での操作に使用されます。Javaのビット演算子には、以下のものがあります。

    • ビット論理積（&）
    • ビット論理和（|）
    • ビット排他的論理和（^）
    • ビット否定（~）

    ビット演算子は、数値のビット表現を操作するために使用されます。例えば、以下のコードはビット演算子の例です。

    int x = 5; // 00000000 0000 0000 0000 0000 0000 0101
int y = 3; // 00000000 0000 0000 0000 0000 0000 0011
int z = x & y; // zは1 (ただし、実際の結果はxとyの共通するビットのみが1になる)

    ビット演算子は、ビット単位の操作に使用されるため、数値の計算とは異なる動作をします。ビット演算子を使用する際には、ビットの表現や符号に注意する必要があります。

    また、サーキット演算子との違いとして、真偽値の評価方法があります。

    サーキット演算子は、左辺の値だけで結果が確定する場合に、右辺の評価を行わない短絡評価を行います。一方で、ビット演算子は、両辺の値を常に評価します。例えば、以下のコードはサーキット演算子とビット演算子の違いを示しています。

    boolean a = true;
boolean b = false;
boolean c = a || b; // cはtrue (短絡評価により、bは評価されない)
boolean d = a | b; // dはtrue (両辺が評価される)

    ビット演算子は、両辺の値を常に評価するため、サーキット演算子と異なり、不要な計算が行われる可能性があります。

    代入演算子

    代入演算子は、変数に値を代入するために使用されます。Javaの代入演算子には、以下のものがあります。

    • 単純代入（=）
    • 加算代入（+=）
    • 減算代入（-=）
    • 乗算代入（*=）
    • 除算代入（/=）
    • 剰余代入（%=）
    • ビット論理積代入（&=）
    • ビット論理和代入（|=）
    • ビット排他的論理和代入（^=）
    • 左シフト代入（<<=）
    • 右シフト代入（>>=）
    • 論理右シフト代入（>>>=）

    代入演算子は、変数に値を代入するために使用されますが、加算代入や減算代入などの複合代入演算子は、変数の現在の値を基に新しい値を計算してから代入します。例えば、以下のコードは複合代入演算子の例です。

    int x = 5;
x += 3; // xは8 (x = x + 3と同じ意味)

    複合代入演算子を使用することで、コードの冗長性を減らし、可読性を向上させることができます。ただし、複合代入演算子を使用する際には、変数の現在の値に基づいて新しい値が計算されることに注意する必要があります。

    三項演算子

    三項演算子は、条件式を評価して、真の場合と偽の場合の値を選択するために使用されます。Javaの三項演算子は、以下の形式で使用されます。

    （条件式） ? （真の場合の値） : （偽の場合の値）

    三項演算子は、条件式を評価して、真の場合には第一の値を返し、偽の場合には第二の値を返します。例えば、以下のコードは三項演算子の例です。

    int x = 10;
String result = (x > 5) ? "Greater than 5" : "Less than or equal to 5"; // resultは"Greater than 5"

    三項演算子は、条件式を評価して値を選択するために使用されますが、複雑な条件式や長い値を使用すると、コードの可読性が低下する可能性があります。適切なバランスを保ちながら、三項演算子を使用することが重要です。

    その他の演算子

    Javaには、条件演算子（instanceof）やインスタンスチェック演算子などのその他の演算子も存在します。これらの演算子は、特定の状況で使用されることがありますが、基本的な演算子と同様に、適切な使用方法を理解することが重要です。

    基本データ型と参照型の比較

    Javaには、基本データ型と参照型の2種類のデータ型があります。基本データ型は、値を直接格納するためのデータ型であり、参照型は、オブジェクトへの参照を格納するためのデータ型です。

    比較演算子の==を用いる場合は、扱うデータ型によって動作が異なります。基本データ型の場合、値の比較を行いますが、参照型の場合、オブジェクトの参照先を比較します。

    int a = 5;
int b = 5;
boolean result = (a == b); // resultはtrue

String s1 = "Hello";
String s2 = new String("Hello");
boolean result2 = (s1 == s2); // result2はfalse

    上記の例では、基本データ型のintの場合、aとbは同じ値を持っているため、resultはtrueになります。一方で、参照型のStringの場合、s1とs2は別の文字列リテラルを参照しているため、result2はfalseになります。

    このように、基本データ型と参照型の比較には注意が必要です。基本データ型の場合は値の比較を行いますが、参照型の場合はオブジェクトの参照先を比較するため、同じ内容を持つオブジェクトであっても異なる参照先を持つ場合は、==演算子で比較するとfalseになることがあります。

    String型の内容を比較する場合は、equals()メソッドを使用することが一般的です。equals()メソッドは、オブジェクトの参照先ではなく、内容を比較します。例えば、以下のコードはStringクラスのequals()メソッドを使用して文字列の内容を比較しています。

    String s1 = "Hello";
String s2 = new String("Hello");
boolean result = s1.equals(s2); // resultはtrue

    ただし、String型以外の参照型には、equals()メソッドは参照先を比較するため、適切な実装が必要です。

    String型とコンスタントプール

    String型は、Javaの基本的なデータ型の一つですが、特別な扱いを受けることがあります。Javaでは、Stringリテラルはコンスタントプールと呼ばれる特別な領域に格納されます。コンスタントプールは、同じ内容の文字列リテラルが複数存在する場合に、それらを共有するための仕組みです。

    具体的には、新しいStringオブジェクトが作成される場合、コンスタントプールに同じ内容の文字列リテラルが存在するかどうかを確認し、存在する場合はその参照を返します。

    例えば、以下のコードでは、同じ内容の文字列リテラルが複数存在しますが、コンスタントプールによって共有されるため、s1とs2は同じ参照先を持ちます。

    String s1 = "Hello";
String s2 = "Hello"; //すでにコンスタントプールにs1が参照している"Hello"が存在するため、s2はs1と同じ参照先を持ちます。
boolean result = (s1 == s2); // resultはtrue

    このように、Stringリテラルはコンスタントプールによって共有されるため、同じ内容の文字列リテラルは同じ参照先を持ちます。ただし、newキーワードを使用してStringオブジェクトを作成する場合は、コンスタントプールを使用せずに新しいオブジェクトが作成されるため、s1とs2は異なる参照先を持ちます。

    String s1 = "Hello";
String s2 = new String("Hello");
boolean result = (s1 == s2); // resultはfalse

    このように、String型の扱いには注意が必要です。Stringリテラルはコンスタントプールによって共有されるため、同じ内容の文字列リテラルは同じ参照先を持ちますが、newキーワードを使用してStringオブジェクトを作成する場合は、コンスタントプールを使用せずに新しいオブジェクトが作成されるため、異なる参照先を持ちます。
