    配列

    この章では、Javaでの配列について学びます。配列とは、同じ型の複数の値を1つの変数に格納するための構造です。

    Javaには、以下のような配列があります。

    • 一元配列
    • 多次元配列

    以下では、各配列の構文と使用方法について詳しく説明します。

    一元配列

    一元配列は、同じ型の値を1次元で格納する配列です。

    型名[] 配列名;
    型名 配列名[];

    どちらの構文も同じ意味を持ちますが、Javaでは、前者の構文が一般的に使用されます。

    配列を宣言しただけでは、配列は作成されません。配列を作成するには、new演算子を使用します。

    配列名 = new 型名[要素数];

    配列を宣言と同時に作成することもできます。

    型名[] 配列名 = new 型名[要素数];

    配列の要素には、インデックスを使用してアクセスします。インデックスは0から始まります。

    配列名[インデックス]

    配列の要素には、インデックスを使用して値を代入することもできます。

    配列名[インデックス] = 値;

    配列に格納した値は、インデックスを用いて取得できます。

    型名 変数名 = 配列名[インデックス];

    配列のインデックスは、0から要素数-1までの範囲でなければなりません。範囲外のインデックスを使用すると、ArrayIndexOutOfBoundsExceptionがスローされます。

    配列の要素数は、lengthフィールドを使用して取得できます。

    配列名.length

    配列の宣言時に初期化を行うこともできます。

    型名[] 配列名 = {値1, 値2, 値3};

    この構文では、配列の要素数は初期化子の数に基づいて自動的に決定されます。

    また、初期化を行わなかった場合、配列の要素はその型のデフォルト値で初期化されます。デフォルト値は以下の通りです。

    • 数値型（byte、short、int、long、float、double）：0
    • char型：'\u0000'（null文字）
    • boolean型：false
    • 参照型（クラス、インターフェース、配列など）：null

    配列の要素は、宣言された型と互換性のある値でなければなりません。例えば、int型の配列にはint型の値を格納できますが、String型の値を格納することはできません。

    配列は、同じ型の複数の値を効率的に管理するための便利な構造ですが、配列のサイズは固定されているため、要素数を変更することはできません。要素数を変更する必要がある場合は、ArrayListなどのコレクションを使用することが一般的です。

    多次元配列

    多次元配列は、同じ型の値を複数の次元で格納する配列です。最も一般的な多次元配列は二次元配列ですが、三次元以上の多次元配列も作成できます。

    二次元配列は、行と列の形式で値を格納する配列です。二次元配列の宣言は以下のようになります。

    型名[][] 配列名;
    型名 配列名[][];

    型名[] 配列名[]

    どちらの構文も同じ意味を持ちますが、Javaでは、前者の構文が一般的に使用されます。

    二次元配列を作成するには、new演算子を使用します。

    配列名 = new 型名[行数][列数];

    二次元配列を宣言と同時に作成することもできます。

    型名[][] 配列名 = new 型名[行数][列数];

    二次元配列の要素には、行と列のインデックスを使用してアクセスします。インデックスは0から始まります。

    配列名[行インデックス][列インデックス]

    二次元配列の要素には、行と列のインデックスを使用して値を代入することもできます。

    配列名[行インデックス][列インデックス] = 値;

    二次元配列に格納した値は、行と列のインデックスを用いて取得できます。

    型名 変数名 = 配列名[行インデックス][列インデックス];

    二次元配列の行インデックスは、0から行数-1までの範囲でなければなりません。列インデックスも同様に、0から列数-1までの範囲でなければなりません。範囲外のインデックスを使用すると、ArrayIndexOutOfBoundsExceptionがスローされます。

    二次元配列の行数は、lengthフィールドを使用して取得できます。

    配列名.length

    ここで取得される長さとは、二次元配列の行数です。

    コマンドライン引数

    Javaプログラムは、コマンドライン引数を介して外部から値を受け取ることができます。コマンドライン引数は、mainメソッドの引数として渡されます。

    public static void main(String[] args) {}

    args配列には、実行時に渡されたコマンドライン引数が格納されます。args[0]には最初の引数、args[1]には2番目の引数が格納されます。

    コマンドライン引数は、プログラムの実行時に外部から値を渡すための便利な方法ですが、引数の数や内容を適切に処理する必要があります。例えば、引数が不足している場合や、引数が期待される形式でない場合は、エラーメッセージを表示するなどの対処が必要です。

    拡張for文

    拡張for文は、配列やコレクションの要素を順番に処理するための構文です。以下のように使用します。

    for (型名 変数名 : 配列名) {
    // 処理
}

    拡張for文は、配列やコレクションの要素を順番に処理する際に便利です。ただし、要素のインデックスが必要な場合は、通常のfor文を使用する必要があります。

    また、拡張for文では処理している配列やコレクションの要素を変更することはできません。

    可変長引数

    可変長引数は、メソッドに渡す引数の数を柔軟に指定できる機能です。可変長引数は、配列として扱われます。

    public void printValues(String... values) {
    for (String value : values) {
        System.out.println(value);
    }
}

    このコードでは、printValuesメソッドは任意の数のString型の引数を受け取ります。呼び出し方は以下のようになります。

    printValues("Hello", "World");

    この呼び出しは、printValuesメソッドに2つのString型の引数を渡しています。可変長引数は、引数の数が不定である場合に便利です。
