    クラスとメソッド

    ここからはオブジェクト指向プログラミングとしてクラスとメソッドについて学びます。

    Javaではシステムを管理しやすいように役割ごとに分割し、それぞれを組み合わせてアプリケーションを構築します。この分割したものをオブジェクトといいます。

    Javaでは属性であるフィールドと機能であるメソッドを持つクラスを定義することで、オブジェクトを生成し、管理します。

    また、フィールドとメソッドを合わせてメンバと呼びます。

    例えば、以下のようなクラスを定義できます。

    public class Car {
    // フィールド（属性）
    private String color;
    private int speed;
    // メソッド（機能）
    public void accelerate() {
        speed += 10;
    }
    public void brake() {
        speed -= 10;
    }
}

    このクラスは、車を表すクラスで、色と速度という属性を持ち、加速とブレーキという機能を持っています。

    フィールド

    フィールドは、オブジェクトの属性を表す変数です。上記のCarクラスでは、colorとspeedがフィールドとして定義されています。

    フィールドの構文は以下の通りです。

    [修飾子] 型名 変数名;

    修飾子は、フィールドのアクセスレベルやその他の特性を指定するために使用されます。例えば、privateはフィールドがクラス内でのみアクセス可能であることを示します。

    メソッド

    メソッドは、オブジェクトの機能を表す関数です。上記のCarクラスでは、accelerateとbrakeがメソッドとして定義されています。

    メソッドの構文は以下の通りです。

    [修飾子] 戻り値の型 メソッド名(引数リスト) {
    // メソッドの本体
}

    修飾子は、メソッドのアクセスレベルやその他の特性を指定するために使用されます。例えば、publicはメソッドがどこからでもアクセス可能であることを示します。

    戻り値の型は、メソッドが返す値の型を指定します。返す値がない場合は、voidを使用します。

    引数リストは、メソッドが受け取る引数の型と名前を指定します。引数がない場合は、空の括弧を使用します。

    メソッドの本体は、メソッドが実行するコードを含むブロックです。

    ブロック内ではローカル変数の宣言や代入、別メソッドの呼び出しを行うことができます。

    クラスの定義とオブジェクトの生成

    クラスを定義するには、classキーワードを使用します。クラスの構文は以下の通りです。

    [修飾子] class クラス名 {
    // クラスの本体
}

    修飾子は、クラスのアクセスレベルやその他の特性を指定するために使用されます。例えば、publicはクラスがどこからでもアクセス可能であることを示します。

    クラスの本体は、フィールドやメソッドなどのクラスのメンバを含むブロックです。

    クラスを定義した後、new演算子を使用してオブジェクトを生成できます。オブジェクトの生成の構文は以下の通りです。

    クラス名 変数名 = new クラス名();

    この構文では、クラス名は生成するオブジェクトのクラスを指定し、変数名は生成されたオブジェクトを参照する変数を指定します。

    例えば、Carクラスのオブジェクトを生成するには、以下のようにします。

    Car myCar = new Car();

    このコードは、Carクラスの新しいオブジェクトを生成し、そのオブジェクトをmyCarという変数に参照させています。

    生成されたオブジェクトのフィールドやメソッドにアクセスするには、ドット演算子を使用します。例えば、myCarオブジェクトのcolorフィールドにアクセスするには、以下のようにします。

    String carColor = myCar.color;

    また、myCarオブジェクトのaccelerateメソッドを呼び出すには、以下のようにします。

    myCar.accelerate();

    変数のスコープ

    変数のスコープは、その変数がアクセスできる範囲を示します。Javaでは、変数は宣言されたブロック内でのみアクセス可能です。

    クラスのフィールドは、クラス全体でアクセス可能ですが、ローカル変数は、その変数が宣言されたメソッドやブロック内でのみアクセス可能です。

    例えば、以下のコードでは、フィールドとローカル変数のスコープの違いを示しています。

    public class Example {
    private int field; // フィールド
    public void method() {
        int localVariable = 10; // ローカル変数
        field = 5; // フィールドにアクセス
        System.out.println(localVariable); // ローカル変数にアクセス
    }
    public void anotherMethod() {
        // System.out.println(localVariable); // ローカル変数にはアクセスできない
    }
}

    このコードでは、fieldはクラスのフィールドであり、methodメソッド内でアクセスできますが、anotherMethodメソッド内ではアクセスできません。一方、localVariableはmethodメソッド内のローカル変数であり、methodメソッド内でのみアクセスできます。

    また、変数名にも注意が必要です。同じ名前の変数が異なるスコープで宣言されている場合、内側のスコープの変数が優先して使用されます。

    public class Example {
    private int value; // フィールド
    public void method() {
        int value = 10; // ローカル変数
        System.out.println(value); // ローカル変数が優先される
    }
}

    このコードでは、fieldはクラスのフィールドであり、methodメソッド内でローカル変数valueが宣言されています。methodメソッド内でvalueを参照すると、ローカル変数が優先されて使用されます。

    フィールドの初期値は配列と同様に、フィールドが宣言されたときに自動的に設定されます。

    オーバーロード

    オーバーロードは、同じ名前のメソッドやコンストラクタを複数定義することです。引数の数や型が異なる場合、適切なメソッドやコンストラクタが呼び出されます。

    例えば、以下のコードでは、同じ名前のメソッドが複数定義されています。

    public class Example {
    public void print() {
        System.out.println("No arguments");
    }
    public void print(String message) {
        System.out.println("Message: " + message);
    }
    public void print(int number) {
        System.out.println("Number: " + number);
    }
}

    このコードでは、printメソッドが3つ定義されています。引数の数や型が異なるため、適切なprintメソッドが呼び出されます。

    オーバーロードされたメソッドを呼び出すには、引数の数や型に応じて適切な引数を渡します。例えば、以下のようにします。

    Example example = new Example();
example.print(); // No arguments
example.print("Hello"); // Message: Hello
example.print(42); // Number: 42

    このコードでは、printメソッドが引数の数や型に応じて適切に呼び出されます。

    呼び出される優先度は、引数の数と型に依存します。

    完全一致する者が最も優先され、次に暗黙の型変換が適用されます。その次にBoxing/Unboxingが適用され、最後に可変長引数が適用されます。

    オーバーロードの優先度

    完全一致 > 暗黙の型変換 > Boxing/Unboxing > 可変長引数

    コンストラクタ

    コンストラクタは、クラスのオブジェクトを生成する際に呼び出される特殊なメソッドです。コンストラクタは、クラス名と同じ名前を持ち、戻り値の型を指定しません。

    例えば、以下のコードでは、Carクラスにコンストラクタを追加しています。

    public class Car {
    private String color;
    private int speed;

    public Car(String color, int speed) {
        this.color = color;
        this.speed = speed;
    }

    public void accelerate() {
        speed += 10;
    }

    public void brake() {
        speed -= 10;
    }
}

    このコードでは、Carクラスにコンストラクタが追加されています。コンストラクタは、オブジェクトを生成する際に呼び出され、フィールドの初期化を行います。

    コンストラクタを使用してCarクラスのオブジェクトを生成するには、以下のようにします。

    Car myCar = new Car("Red", 0);

    このコードは、Carクラスの新しいオブジェクトを生成し、そのオブジェクトをmyCarという変数に参照させています。コンストラクタには、colorとspeedの初期値が引数として渡されます。

    コンストラクタは、クラスに複数定義することができます。これをコンストラクタのオーバーロードと呼びます。オーバーロードされたコンストラクタは、異なる引数リストを持ち、オブジェクトの生成時に適切なコンストラクタが呼び出されます。

    public class Car {
    private String color;
    private int speed;
    public Car() {
        this.color = "Unknown";
        this.speed = 0;
    }
    public Car(String color) {
        this.color = color;
        this.speed = 0;
    }
    public Car(String color, int speed) {
        this.color = color;
        this.speed = speed;
    }
    public void accelerate() {
        speed += 10;
    }
    public void brake() {
        speed -= 10;
    }
}

    このコードでは、Carクラスに3つのコンストラクタが定義されています。引数の数や型が異なるため、オブジェクトの生成時に適切なコンストラクタが呼び出されます。

    また、コンストラクタを設定しなかった場合、Javaはデフォルトコンストラクタを自動的に生成します。

    staticメンバとインスタンスメンバ

    staticメンバは、クラスのインスタンスを作成せずにアクセスできるメンバーです。一方、インスタンスメンバは、クラスのインスタンスを作成してからアクセスできるメンバーです。

    staticメンバは、クラス全体で共有されるため、すべてのインスタンスから同じ値を参照します。インスタンスメンバは、各インスタンスごとに異なる値を持つことができます。

    staticメンバを定義するには、staticキーワードを使用します。例えば、以下のコードでは、Carクラスにstaticフィールドとstaticメソッドが追加されています。

    public class Car {
    private String color;
    private int speed;
    private static int numberOfCars = 0; // staticフィールド
    public Car(String color, int speed) {
        this.color = color;
        this.speed = speed;
        numberOfCars++; // コンストラクタでstaticフィールドを更新
    }
    public void accelerate() {
        speed += 10;
    }
    public void brake() {
        speed -= 10;
    }
    public static int getNumberOfCars() { // staticメソッド
        return numberOfCars;
    }
}

    このコードでは、CarクラスにnumberOfCarsというstaticフィールドとgetNumberOfCarsというstaticメソッドが追加されています。numberOfCarsは、Carクラスのインスタンスが生成されるたびに更新されます。getNumberOfCarsメソッドは、Carクラスのインスタンスを作成せずに呼び出すことができます。

    staticメンバにアクセスするには、クラス名を使用します。例えば、CarクラスのnumberOfCarsフィールドにアクセスするには、以下のようにします。

    int totalCars = Car.numberOfCars;

    CarクラスのgetNumberOfCarsメソッドを呼び出すには、以下のようにします。

    int totalCars = Car.getNumberOfCars();

    インスタンスメンバにアクセスするには、クラスのインスタンスを使用します。例えば、myCarオブジェクトのcolorフィールドにアクセスするには、以下のようにします。

    String carColor = myCar.color;

    myCarオブジェクトのaccelerateメソッドを呼び出すには、以下のようにします。

    myCar.accelerate();

    staticメンバはインスタンスからアクセスすることもできますが、通常はクラス名を使ってアクセスします。

    また、staticメンバとインスタンスメンバの間には、アクセスに関するルールがあります。

    staticメンバ同士、またはインスタンスメンバ同士では、通常のアクセス規則が適用されます。

    インスタンスメンバからstaticメンバへアクセスすることはできます。

    しかし、staticメンバからインスタンスメンバへアクセスすることはできません。

    staticメンバからインスタンスメンバへアクセスするには、クラスのインスタンスを作成してからアクセスする必要があります。

    public class Example {
    private int instanceField; // インスタンスメンバ
    private static int staticField; // staticメンバ
    public static void staticMethod() {
        // instanceField = 10; // エラー: staticメソッドからインスタンスフィールドにアクセスできない
        Example example = new Example(); // クラスのインスタンスを作成
        example.instanceField = 10; // インスタンスを通じてアクセス
    }
    public void instanceMethod() {
        instanceField = 10; // インスタンスメンバにアクセス
        staticField = 20; // staticメンバにアクセス

        }
    }

    このコードでは、staticMethodはstaticメソッドであり、instanceFieldはインスタンスメンバです。staticMethod内でinstanceFieldに直接アクセスすることはできませんが、Exampleクラスのインスタンスを作成してからアクセスすることができます。

    staticメンバはnullを代入した変数に対してもアクセスできます。

    public class Example {
    private static int staticField; // staticメンバ
    public static void staticMethod() {
        Example example = null; // nullのインスタンス
        example.staticField = 10; // nullを通じてstaticメンバにアクセス
        }
        }

    このコードでは、exampleという変数はnullを参照していますが、staticFieldにアクセスすることができます。これは、staticメンバがクラス全体で共有されるため、クラスの情報を参照してアクセスできるからです。

    staticイニシャライザ

    staticイニシャライザは、クラスが初めてロードされたときに実行されるコードブロックです。staticイニシャライザは、staticフィールドの初期化やその他のクラスレベルの初期化を行うために使用されます。

    staticイニシャライザの構文は以下の通りです。

    static {
    // staticイニシャライザのコード
}

    このコードブロック内のコードは、クラスが初めてロードされたときに一度だけ実行されます。staticイニシャライザは、クラスのstaticフィールドを初期化するために便利です。

    アクセス修飾子

    アクセス修飾子は、クラスやクラスのメンバのアクセスレベルを指定するために使用されます。Javaには、public、protected、private、およびデフォルト（アクセス修飾子なし）の4つのアクセス修飾子があります。

    publicは、クラスやメンバがどこからでもアクセス可能であることを示します。protectedは、クラスやメンバが同じパッケージ内のクラスやサブクラスからアクセス可能であることを示します。privateは、クラスやメンバがクラス内でのみアクセス可能であることを示します。デフォルトは、クラスやメンバが同じパッケージ内のクラスからアクセス可能であることを示します。

    アクセス修飾子は、クラスやメンバの宣言の前に指定します。例えば、以下のコードでは、Carクラスのフィールドとメソッドにアクセス修飾子が指定されています。

    public class Car {
    private String color; // privateフィールド
    private int speed; // privateフィールド
    public Car(String color, int speed) { // publicコンストラクタ
        this.color = color;
        this.speed = speed;
    }
    public void accelerate() { // publicメソッド
        speed += 10;
    }
    public void brake() { // publicメソッド
        speed -= 10;
    }
}

    このコードでは、colorとspeedフィールドはprivateで宣言されており、Carクラスの外部から直接アクセスすることはできません。一方、コンストラクタとメソッドはpublicで宣言されており、Carクラスの外部からアクセスすることができます。

    アクセス修飾子を適切に使用することで、クラスの設計をより安全で柔軟にすることができます。例えば、フィールドをprivateにすることで、クラスの内部状態を外部から直接変更できないようにし、メソッドをpublicにすることで、クラスの機能を外部から利用できるようにすることができます。これをカプセル化と言います。

    また、コンストラクタ、フィールド、メソッドにはpublic, protected, private, およびデフォルト（アクセス修飾子なし）の4つのアクセス修飾子が使えますが、クラスに関してはpublicおよびデフォルトのアクセス修飾子のみ使用できる点に注意してください。

    値コピーと参照コピー

    Javaでは、変数に値を代入するとき、基本データ型の場合は値がコピーされますが、参照データ型の場合は参照がコピーされます。

    基本データ型の変数に値を代入すると、その値がコピーされます。例えば、以下のコードでは、aとbは独立した変数であり、aの値を変更してもbの値には影響しません。

    int a = 5;
int b = a; // aの値がbにコピーされる
a = 10; // aの値を変更
System.out.println(b); // bの値は5のまま

    一方、参照データ型の変数に値を代入すると、その変数はオブジェクトへの参照をコピーします。例えば、以下のコードでは、array1とarray2は同じ配列を参照しており、array1の内容を変更するとarray2の内容も変更されます。

    int[] array1 = {1, 2, 3};
int[] array2 = array1; // array1の参照がarray2にコピーされる
array1[0] = 10; // array1の内容を変更
System.out.println(array2[0]); // array2の内容も変更される（10と表示される）

    このように、基本データ型の変数は値がコピーされるため、独立した変数になりますが、参照データ型の変数は参照がコピーされるため、同じオブジェクトを参照することになります。

    メソッドの引数についても同様です。基本データ型の引数は値が渡され、参照データ型の引数は参照が渡されます。

    例えば、以下のコードでは、基本データ型の引数を変更しても呼び出し元の変数には影響しませんが、参照データ型の引数を変更すると呼び出し元のオブジェクトも変更されます。

    public class Example {
    public void modifyPrimitive(int value) {
        value = 10; // 基本データ型の引数を変更
    }
    public void modifyReference(int[] array) {
        array[0] = 10; // 参照データ型の引数を変更
    }
}
Example example = new Example();
int primitiveValue = 5;
example.modifyPrimitive(primitiveValue);
System.out.println(primitiveValue); // primitiveValueは5のまま
int[] referenceValue = {1, 2, 3};
example.modifyReference(referenceValue);
System.out.println(referenceValue[0]); // referenceValueの内容が変更される（10と表示される）

    このコードでは、modifyPrimitiveメソッドは基本データ型の引数valueを変更しますが、呼び出し元のprimitiveValueには影響しません。一方、modifyReferenceメソッドは参照データ型の引数arrayを変更するため、呼び出し元のreferenceValueも変更されます。

    ガベージコレクション

    ガベージコレクションは、Javaのメモリ管理機能の一部であり、不要になったオブジェクトを自動的に回収する仕組みです。ガベージコレクションは、プログラムが使用しているメモリを効率的に管理し、メモリリークを防止するために重要な役割を果たします。

    Javaのガベージコレクションは、オブジェクトが参照されなくなったときに、そのオブジェクトを自動的に回収します。例えば、以下のコードでは、myCarオブジェクトが参照されなくなった後、ガベージコレクションによって回収される可能性があります。

    Car myCar = new Car("Red", 0);
myCar = null; // myCarオブジェクトへの参照を解除
// ガベージコレクションがmyCarオブジェクトを回収する可能性がある

    ガベージコレクションは、Javaのヒープメモリを管理するために使用されます。ヒープメモリは、オブジェクトが動的に割り当てられる領域であり、ガベージコレクションによって不要なオブジェクトが回収されることで、ヒープメモリの効率的な利用が促進されます。

    ガベージコレクションの対象になるのはnull代入された変数や別のオブジェクトの参照により到達不可能になったオブジェクトであることを覚えておきましょう。

    //nullを代入してガベージコレクションの対象にする
myCar = null;
//別変数を参照して到達不可能にする
Car anotherCar = new Car("Blue", 0);
myCar = anotherCar; // myCarはanotherCarを参照するようになるため、元のmyCarオブジェクトは到達不可能になる

    ガベージコレクションは、Javaのメモリ管理を自動化するための重要な機能ですが、ガベージコレクションがいつ実行されるかは予測できません。そのため、不要なオブジェクトへの参照を解除することや、適切なデータ構造を使用することなど、メモリ管理に注意を払うことが重要です。
